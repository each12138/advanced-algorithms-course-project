## 主题：FLP 不可能性结果论文精读

### 阅读论文
- 文件：`papers/1985-flp_更清晰版本.pdf`
- Fischer, Lynch, Paterson (1985) — "Impossibility of Distributed Consensus with One Faulty Process"

---

### 讨论内容总结

#### 1. Run（运行）vs Deciding Run（判定运行）

| 概念 | 定义 |
|------|------|
| **Run（运行）** | 从某个初始配置出发，按一个调度（schedule）依次执行事件后得到的步骤序列。即协议实际执行的一条完整路径。 |
| **Deciding Run（判定运行）** | 在该运行中，**至少有一个进程**进入了判定状态（decision state），输出了 0 或 1。 |
| **关键区别** | Run 不一定有结果；Deciding Run 一定有人做出了判定。 |

**在证明中的角色**：
- 完全正确（totally correct）要求：**每个**可接受运行（admissible run）都必须是 deciding run
- FLP 反证法：构造一个可接受运行**不是** deciding run → 矛盾

---

#### 2. Initial Configuration（初始配置）vs Configuration（配置）

| 概念 | 定义 |
|------|------|
| **Initial Configuration** | 每个进程都处于初始状态（input register 持有输入值）+ **消息缓冲区为空** |
| **Configuration** | 经过任意步事件后到达的系统状态，进程状态已变，缓冲区可能有消息 |

**关键性质**：
- 初始配置 + 一个 step → **普通配置**，不是初始配置
- 初始配置 + 一个 run → **普通配置**，不是初始配置（除非 run 为空）
- FLP 论文是 **单轮（one-shot）共识模型**，没有"下一轮"的概念
- 引理 2：存在一个双值的**初始配置**

**与博客描述的区别**：
- FLP 论文：initial configuration 是**唯一**的起点，由 input vector 给定
- 有些博客描述的是多轮模型（如 Paxos/Raft 每轮一个实例），将"每轮的起点"称为 initial configuration
- 读论文时应严格遵循论文的原始定义

---

#### 3. 判定值与一致性

- 论文定义：如果配置 `C` 中**某个进程**处于判定状态且输出值为 `v`，则称配置 `C` 具有判定值 `v`
- 这是**基于一致性保证的定义**：没有可达配置能拥有多于一个的判定值
- 一个进程判定 → 全局判定值已锁定（由于一致性），但不保证所有非故障进程都能学到这个值
- 论文要求的终止性只是"每个可接受运行中至少有一个进程判定"，而不是"所有非故障进程都判定"

---

#### 4. 核心证明逻辑

1. **引理 2**：存在双值的初始配置
2. **引理 3**：从双值配置出发，总能找到下一个双值配置
3. **构造**：从双值初始配置开始，反复应用引理 3，构造一个无限的可接受运行，永远停留在双值状态
4. **矛盾**：这个 run 是 admissible 但不是 deciding → 与"完全正确"的要求矛盾
5. **结果**：不存在能容忍一个故障的完全正确的异步共识协议
