### 讨论内容总结

#### Q1: 异步系统的定义是什么？

**我的问题**：文中对异步系统的定义是什么？

**AI 回答要点**：
- 核心假设：没有任何关于时间的信息可用
- 四条含义：(1) 无消息延迟上界；(2) 无进程速度假设；(3) 无同步时钟；(4) 无法检测进程死亡
- 系统演化完全由事件顺序（schedule）而非时间决定

#### Q2: Internal State 和 Configuration 的定义与关联

**我的问题**：这两个概念的定义和关联是什么？

**AI 回答要点**：
- Internal State 是局部（单个进程的 input register + output register + program counter + internal storage）
- Configuration 是全局（所有进程的 internal state + message buffer）
- Configuration 是全局快照，Internal State 是组成部件
- 二者区分是 Lemma 1（调度交换性）的基础——不相交进程的步互不影响

#### Q3: Partially correct / Totally correct / Bivalent / Univalent 的定义

**我的问题**：这四个概念的定义分别是什么？

**AI 回答要点**：
- **Partially correct**：(1) 无 configuration 同时包含两种决议值 (2) 两种决议值（0 和 1）都有可能被达到
- **Totally correct in spite of one fault**：部分正确 + 每个 admissible run 都是 deciding run
- **Bivalent**：从该配置出发，两种决议值都可达，系统尚未锁定方向
- **Univalent**：从该配置出发，只有一种决议值可达，命运已定

#### Q4: 论文模型 = 分布式环境 + 消息系统？

**我的理解**：论文建立起的模型相当于分布式环境加上消息系统。

**AI 补充/修正**：
- 基本正确。更精确地说：一组确定型自动机 + 一个异步消息系统
- 进程是确定型的，非确定性唯一来源是消息系统
- 两个层面通过事件 e = (p, m) 交互

#### Q5: Lemma 3 的含义理解

**我的理解**：从一个 bivalent 的 configuration A 出发，有一个事件 e，一定存在另一个 bivalent 的 configuration B，在 A 变换到 B 的过程中可以把 e 放在最后一步。

**AI 修正**：
- 基本正确，但需注意 Lemma 3 是存在性断言（∃），不是全称（∀）——不是随便选一条路都能把 e 放最后，而是存在一条路能这样绕
- 路径是：A →（不含 e 的步骤）→ C' → e → B（bivalent）
- 这个存在性恰好够 Theorem 1 的构造使用——每一轮只需要找出一条绕路的方法

#### Q6: Lemma 3 证明中情况 2 的详细推导

**我的问题**：再详细解释一下 Lemma 3 证明过程中的情况 2。

**AI 回答要点**：
- 情况 2 中 e' = (p, m')，和 e 是同一个进程的事件，Lemma 1 用不了
- 通过构造一个不含 p 的 deciding run σ，利用 Lemma 1 的交换性和 write-once 性质，使得同一个决议值 v 被同时钳制在 0 和 1 上，导出矛盾
- 关键依赖：(1) 异步系统允许无限期不调度 p (2) Lemma 1 保证 σ 和 e/e' 可交换 (3) output register 是 write-once 的

#### Q7: Theorem 1 的准确含义

**我的理解**：不存在一个共识协议能够保证在允许最多单进程故障的情况下正确。

**AI 修正**：
- 如果"正确"指的是"部分正确 + 总能终止"，那就对
- 如果"正确"只指"部分正确"，那不对——部分正确的协议是存在的
- 更准确的表述：在异步系统中，不存在一个共识协议能同时保证一致性、有效性和终止性，哪怕只允许一个进程崩溃

#### Q8: 关于 bivalent 之间反复的理解

**我的理解**：也就是说无法保证每一个 admissible run 都能做出决策？也就是会在 bivalent 的 configuration 之间反复？

**AI 确认**：
- 对，就是这个意思
- 证明的构造就是 Lemma 2 找到 bivalent 起点，Lemma 3 反复绕到下一个 bivalent，无限循环，永不 deciding
- Theorem 1 的结论就是总存在一条 admissible 路径让系统永远不做决定

#### Q9: Theorem 2 的协议原理

**我的问题**：Theorem 2 是如何通过构建有向图证明该协议可行性的？

**AI 回答要点**：
- 条件：执行中无进程死亡，严格多数初始存活
- 阶段一：构建有向图 Γ，每个节点入度 k-1（多数）
- 阶段二：构建传递闭包 Γ⁺，共享祖先信息和初始值
- 阶段三：找出唯一初始团（大小 ≥ k），用约定规则做相同决议
- 与 Theorem 1 的关系：故障时机是关键——"一开始就死了"和"跑到一半死了"是两回事
