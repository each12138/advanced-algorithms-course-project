### 讨论内容总结

#### Q1: 异步系统的定义

**我的提问**：文中对异步系统的定义是什么？

**AI 回答要点**：
- 核心假设：没有任何关于时间的信息可用
- 四条具体含义：无消息延迟上界、无进程速度假设、无同步时钟、无法检测进程死亡
- 系统演化完全由事件顺序（schedule）而非时间决定

#### Q2: Internal State 与 Configuration 的定义与关联

**我的提问**：这两个概念的定义是什么？有什么关联？

**AI 回答要点**：
- Internal State 是局部（单个进程），Configuration 是全局（所有进程 + 消息缓冲区）
- Configuration 是快照，Internal State 是组成部件
- 二者区分是 Lemma 1（调度交换性）的基础

#### Q3: Partially correct / Totally correct / Bivalent / Univalent 的定义

**我的提问**：文中对这几个概念的定义是什么？

**AI 回答要点**：
- Partially correct：(1) 不矛盾 (2) 两种值都可能
- Totally correct：部分正确 + 每个 admissible run 都是 deciding run
- Bivalent：从该配置出发两种决议值都可达
- Univalent：从该配置出发只有一种决议值可达
- 四个概念构成 FLP 证明的骨架

#### Q4: 论文模型的整体结构

**我的理解**：论文建立起的模型相当于分布式环境加上消息系统。

**AI 补充/修正**：
- 基本正确，更精确地说是"一组确定型自动机 + 一个异步消息系统"
- 进程是确定型的，唯一的非确定性来源是消息系统
- 两个层面通过事件 e = (p, m) 交互
